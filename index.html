5LB Catalog API

Публичные эндпоинты каталога товаров 5lb.pro. Только GET, авторизация не требуется, ответы в формате JSON с полями в camelCase.

GET

Получить каталог

https://mobile.5lb.pro/api/v1/catalog-5lb

Список товаров с пагинацией, фильтрацией по категориям, бренду и стране, а также полнотекстовым поиском по названию, бренду и артикулам.

GET

Категории

https://mobile.5lb.pro/api/v1/catalog-5lb/sections

Возвращает уникальные значения section1…section4 для построения дерева категорий.

Пример запроса
curl https://mobile.5lb.pro/api/v1/catalog-5lb/sections
const sections = await fetch("https://mobile.5lb.pro/api/v1/catalog-5lb/sections")
  .then(r => r.json());
Пример ответа
{
  "section1": ["Спортивное питание", "Витамины и минералы", "Здоровое питание"],
  "section2": ["Аминокислоты", "Протеин", "Предтренировочные"],
  "section3": ["BCAA", "Глютамин"],
  "section4": []
}
GET

Товар по артикулу

updated
https://mobile.5lb.pro/api/v1/catalog-5lb/by-article/:articleMain

Один articleMain может соответствовать нескольким вариантам товара (разные вкусы / фасовки) — ответ всегда массив items.

что нового
Ответ дополнительно содержит imageWidth и imageHeight — реальные размеры изображения в пикселях. Если фото не загружено, оба поля равны null.

Параметр

required
string

GET

Товар по ID

https://mobile.5lb.pro/api/v1/catalog-5lb/:id

Возвращает один объект товара по внутреннему CUID. Если не найден — 404 Not Found.

Path-параметр
Параметр
Тип
Описание
id
required
string
Уникальный CUID (поле id из любого ответа каталога).
Пример запроса
curl https://mobile.5lb.pro/api/v1/catalog-5lb/cmmbpn9g6000cc2opmo8shz83
Пример ответа
{
  "id":          "cmmbpn9g6000cc2opmo8shz83",
  "productName": "Whey Gold Standard",
  "brand":       "Optimum Nutrition",
  "articleMain": "1010002",
  "imageUrl":    "https://mobile.5lb.pro/images/catalog/1010002-CLYLW-VOL700.png",
  // ... полный объект товара
}

media

Статические изображения товаров (раздаются nginx).

URL

Изображение товара

https://mobile.5lb.pro/images/catalog/{articleMain}-{articleAdditional}.png
формат имени файла
{articleMain}-{articleAdditional}.{ext} — если доп. артикул есть
{articleMain}.{ext} — если доп. артикул пустой

В ответах API поле imageUrl уже содержит готовый полный URL или null. Эндпоинт /by-article также отдаёт imageWidth и imageHeight.
Примеры URL
# с доп. артикулом
https://mobile.5lb.pro/images/catalog/1010002-CLYLW-VOL700.png

# без доп. артикула
https://mobile.5lb.pro/images/catalog/1010002.png
